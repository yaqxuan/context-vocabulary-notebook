# Security Policy

## Supported Versions

当前项目仍处于早期版本。安全修复优先面向 `main` 分支最新版本。

| Version | Supported |
|---------|-----------|
| `main` | Yes |
| Older commits / forks | No |

## Reporting a Vulnerability

如果你发现安全漏洞，请不要创建公开 Issue，也不要公开发布可复现攻击细节。

推荐报告方式：

1. 使用 GitHub 的私有漏洞报告 / Security Advisory 功能（如果仓库已启用）。
2. 如果私有漏洞报告不可用，请通过仓库维护者公开的 GitHub 联系方式联系维护者，并只发送必要摘要；等维护者确认私密沟通渠道后，再发送完整细节。

报告时请尽量包含：

- 漏洞类型和影响范围。
- 复现步骤。
- 受影响版本或 commit。
- 相关日志、截图或最小复现代码。
- 你认为可行的修复建议。

## Response Expectations

维护者会尽力：

- 确认报告是否收到。
- 评估漏洞影响。
- 在修复前避免公开细节。
- 修复完成后在合适范围内说明影响和处理方式。

由于这是个人维护项目，响应时间可能受维护者时间影响。

## Security Scope

重点安全范围包括：

- 本地文件上传、媒体访问和路径处理。
- 导入导出 ZIP 处理。
- 本地 SQLite 数据处理。
- AI API Key 的本地保存、遮罩显示和导出排除。
- API 输入校验和错误处理。

以下通常不视为安全漏洞：

- 需要用户主动修改本地源码或本地数据库才能触发的问题。
- 已获得用户本机权限后的本地文件读取。
- 第三方服务、浏览器扩展、系统包管理器或 npm registry 本身的安全问题。

## Dependency Security

项目依赖 npm 包和系统级工具。贡献者和用户应自行评估第三方依赖风险，并在安装脚本执行前理解其会安装 Git、Node.js、native build tools 等外部软件。

如发现依赖漏洞，请提交私密报告或 PR，说明受影响包、影响范围和建议升级版本。
